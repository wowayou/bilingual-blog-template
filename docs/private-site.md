# Private Site From Public Template

Keep the public template generic. Use a private repository for personal content and visual identity.

## Recommended Flow

1. Finish and verify the public template.
2. Push it to a public GitHub repository.
3. Mark the repository as a GitHub Template repository.
4. Create a private repository from that template.
5. Change personal content, site profile data, and visual tokens only in the private repository.

## What Belongs in the Private Site

- Real author name and bio
- Real domain and analytics choices
- Personal color tokens, typography, and homepage tone
- Drafts, unpublished ideas, and private writing workflow
- Giscus repository settings

## What Should Stay in the Template

- Bilingual routing
- Content collection schema
- Search, RSS, sitemap, and deployment workflow
- Theme system
- Generic Markdown styling
- Documentation for starting from scratch
